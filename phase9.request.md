# Phase 10 웹 푸시 알림 — 구현 요청

## 배경

plan.md에 Phase 10으로 웹 푸시 알림 기능의 조사·설계가 이미 완료되어 있고,
`refreshMarketData`에 연동 지점(`evaluateAlertsHook`)까지 빈 훅으로 미리 만들어져
있는 상태다. 이제 실제 구현을 진행하고 싶다.

## 요청

코드 작성 전에 먼저 확인하고 계획을 보고해줘:

1. **기존 설계 재확인** — plan.md의 Phase 10 설계 내용을 다시 훑어보고,
   그동안(Phase 11~18) 아키텍처가 여러 번 바뀌었으니(QStash 잡 구조, Redis 키 패턴,
   feeds 파이프라인 등) 그 설계가 지금 코드 기준으로도 여전히 유효한지 확인해줘.
   안 맞는 부분이 있으면 뭐가 바뀌어야 하는지 짚어줘.

2. **알림 대상/조건 정리** — 이 프로젝트에 이미 있는 데이터(보유·관심종목,
   당일 등락률, 공시, 뉴스 등) 중 어떤 이벤트를 푸시 알림으로 보낼 수 있는지
   후보를 정리해줘. (예: 보유종목 급등락, 관심종목 신규 공시, 등록가 승격 등 —
   기존에 설계했던 내용이 있으면 그걸 기준으로, 없으면 새로 제안)

3. **구현 범위 제안** — 웹 푸시는 브라우저 권한 요청, 구독 관리(PushSubscription
   저장), 서비스 워커, 실제 발송 로직 등 여러 단계로 이뤄지는데, 한 번에 다
   만들지 단계별로 나눌지 제안해줘. (이 프로젝트가 지금까지 큰 기능은 보통
   단계별로 쪼개서 진행해온 패턴을 참고)

4. **기존 골격과의 통합** — `evaluateAlertsHook`이 이미 `refreshMarketData`에
   심어져 있다고 했는데, 이걸 그대로 살려 쓸지, 아니면 `refreshFeeds` 같은
   다른 잡과도 연동이 필요한지(공시·뉴스 알림까지 하려면) 검토해줘.

## 진행 방식

계획을 먼저 보고하고, 승인받은 뒤 구현 시작할 것. 계획 안에 단계 구분(예: 1단계
구독 등록까지, 2단계 실제 알림 발송)이 있으면 각 단계마다 확인받고 다음 단계로
넘어가는 방식으로 진행해줘.
