# 화면 확인 중 발견된 수정사항 3건

## 1) 핫종목 카드 — 일일 등락률 순위 100위

진행 승인. 상세 설계(Redis 키 구조·UI 배치)까지 잡아서 계획을 먼저 보고해줘.
코드 작성 전에 계획 승인받고 진행할 것.

(참고: 조사 결과 KIS 등락률 순위 API(FHPST01700000)가 서버에서 이미 정렬해서
제공하므로, Phase 14 월간 핫종목 같은 전체 종목 스캔 없이 호출 4회 수준으로 가능함을
확인함. 다만 당일 등락률은 장중 실시간으로 계속 바뀌므로, 월간 잡이 아니라 기존
시세 갱신 잡에 얹어 `market:dailyFluctuation` 같은 신규 키에 스냅샷 저장하고,
핫종목 페이지에 "당일 등락률" 탭을 신설하는 구조로 제안된 바 있음.)

## 2) 고정 주간(월~금) 등락률 순위 100위 — 조사만 먼저

실시간이 아니라 **매주 한 번(예: 금요일 장 마감 후)만 계산하면 되는 배치성 데이터**로
"지난 주간 등락률 순위 100위"를 만들 수 있는지 조사해줘. 코드는 작성하지 말 것.

확인할 것:
- 방금 찾은 KIS 등락률 순위 API(FHPST01700000)에 기간 구분(일/주/월) 옵션이 있는지.
  있으면 일일 등락률과 비슷하게 저렴하게 구현 가능
- 옵션이 없으면: Phase 14 월간 핫종목처럼 전체 종목을 조회해서 직접 계산해야 하는데,
  이건 주 1회만 실행하면 되는 배치 작업이라 실시간 부담 없이 Phase 14와 같은 방식
  (50초 예산·커서 기반 재개 처리)으로 처리 가능한지 검토

조사 결과와 실현 가능성부터 보고할 것. 진행 여부는 보고 받은 뒤 결정.

## 3) 보유종목·관심종목 등록 — 종목명 검색으로 변경 (조사만 먼저)

지금은 종목 등록 시 종목코드를 직접 입력해야 하는 방식인데, **종목명을 입력하면
매칭되는 종목들이 검색되고, 검색 결과를 터치하면 그 종목이 선택되는 기능**을
만들고 싶다. 코드는 작성하지 말고 아래를 조사해서 계획을 보고해줘.

확인할 것:
- 종목코드↔종목명 전체 매핑 데이터를 어디서 가져올 수 있는지
  (KIS API에 전체 종목 마스터 제공 엔드포인트가 있는지, 아니면 Phase 17-1에서 만든
  DART corpCode 매핑을 응용할 수 있는지)
- 검색을 서버에서 처리할지, 아니면 전체 목록을 미리 받아서 클라이언트에서
  필터링할지 — 데이터 규모를 보고 판단해서 제안
- 기존 등록 폼(종목코드+매입금액 입력) UI를 어떻게 바꿀지 간단히 제안

조사 결과와 계획부터 보고할 것. 진행 여부는 보고 받은 뒤 결정.
