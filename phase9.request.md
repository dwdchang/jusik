# 문서 역할 분담 명문화

## 배경
plan.md는 Phase 단위 이력 문서, research.md는 구조 기준(§1~10)
스냅샷 문서라 조직 원리가 서로 다르다. 이 차이가 명시돼 있지
않으면 앞으로도 "새 조사=새 섹션 추가" 같은 착오가 반복될 수
있어, 각 문서에 역할을 명확히 기록해둔다.

## 요구사항

1. **research.md**
   - 문서 최상단(목적 설명 부근)에 아래 내용을 추가한다:
     - "이 문서는 Phase 순서가 아니라 프로젝트 구조 기준 §1~10으로
       조직된다. 시간 순서와 무관하게 항상 '현재 코드 기준 최신
       상태'만 담는다. 새 조사 내용은 새 번호 섹션을 추가하지 않고
       해당하는 기존 §1~10 절에 통합한다."
     - plan.md와의 역할 차이도 한 줄로 명시:
       "plan.md는 Phase 단위 작업 이력(시간순, 의사결정 기록)이고,
       research.md는 현재 코드 구조 스냅샷(시간 무관, 항상 최신
       유지)이다 — 두 문서는 조직 원리가 다르므로 혼동하지 말 것."

2. **plan.md**
   - 문서 최상단(또는 §5 참조 문서 절 부근)에 아래 내용을 추가한다:
     - "plan.md는 Phase 단위로 시간순 작업 이력을 기록하는 문서다.
       research.md(구조 기준 §1~10, 항상 최신 유지)와는 조직
       원리가 다르므로, 코드 구조 조사 내용을 여기에 Phase로
       추가하지 말고 research.md에 반영할 것."

3. AGENTS.md의 §5(참조 문서) 절에도 이 역할 구분을 한 줄로 요약해
   추가한다 (이미 있다면 중복 추가하지 않는다):
   - "plan.md — Phase 단위 작업 이력. research.md — 현재 코드
     구조 스냅샷(§1~10, 시간 무관, 항상 최신 유지)."

## 완료 조건
- research.md, plan.md, AGENTS.md 세 파일 모두에 위 역할 구분이
  명시됨
- 기존 내용은 변경하지 않고 안내 문구만 추가
- 세 파일 diff를 보여주면 확인 후 커밮 지시할게
