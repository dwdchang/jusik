# Phase 15 - 관심종목 + 시장 통합 기능

Phase 14(핫종목)와 별개로, 앞서 논의했던 두 기능을 조사·설계한다.
코드는 아직 작성하지 않는다.

## 1. 시장 항목 (원달러+미국금리+유가 통합)

- 기존 "원달러 환율", "미국 10년물 금리" 두 개의 별도 홈 카드를
  "시장"이라는 하나의 카드로 통합
- 여기에 유가(원유)도 추가
- 홈 카드: 대표값 하나만 표시 (예: 원달러 + 등락률)
- 상세 페이지(/indices/market): 원달러/미국금리/유가를
  각각 미니 카드나 리스트 형태로 나열, 각각 미니 차트 포함

조사 필요:
- 유가(원유 등)가 KIS의 해외지수/환율 조회 엔드포인트
  (FHKST03030100)로 조회 가능한지, 종목코드가 무엇인지 실측 확인
- 기존 원달러/금리 데이터를 Phase 11의 market:detail:* 저장 구조에
  어떻게 통합할지 (키 구조 변경 필요 여부)

## 2. 관심종목

- 보유종목과 유사하지만 수량 없이, 종목코드만 등록
- 데이터 모델: Watchlist { symbolCode, name, registeredAt(등록일),
  priceAtRegistration(등록 시점 가격) }
- 등록 시점은 나중에 수정 가능 (수정 시 그 날짜의 종가를
  다시 조회해서 priceAtRegistration 갱신)
- 수익률 계산: (현재가 - 등록시점가격) / 등록시점가격 × 100
  (수량/금액 개념 없이 순수 비율만)
- 상세 페이지: 보유종목 상세 페이지와 동일한 정보블록(시총/
  배당/실적/투자지표) + 차트 재사용 (Phase 13에서 만든
  종목 상세 컴포넌트 재사용 가능한지 확인)
- 홈 카드: 관심종목 개수 또는 평균 수익률 요약 표시
  (형태는 설계 시 제안)
- 로그인 필요 (보유종목과 동일한 보안 수준, Redis 키는
  watchlist:{email} 형태로 암호화 적용 여부도 검토 -
  Phase 12 암호화 대상에 포함할지 판단)
- 갱신은 Phase 11의 기존 갱신 잡(장중 10분/15:40/18:15)에
  보유종목과 함께 얹는 방식으로 통합 가능한지 검토
